# CI Pipeline + Container Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans or subagent-driven-development to implement task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Add a GitHub Actions pipeline that tests every change and publishes a multi-arch container image to GHCR, plus a human-gated `/release` skill (mirroring abs-cli) that cuts versioned releases.

**Architecture:** One workflow (`build.yml`) with a `test` job (format + unit tests) gating an `image` job (buildx multi-arch → GHCR). Tags: `:main` + `:main-<sha>` on main pushes; `:X.Y.Z` + `:latest` on GitHub Releases. The release procedure is a gated Claude skill that bumps version, writes the changelog, PRs for CI, tags, and lets CI publish the release image.

**Tech Stack:** GitHub Actions, docker/build-push-action + buildx + QEMU, GHCR, .NET 10, `dotnet format`.

## Global Constraints

- Generalize the existing reverse-proxy comment in `Program.cs` so it doesn't name a specific proxy.
- Image: `ghcr.io/thomaslazar/inkshelf`, **public** package, platforms `linux/amd64,linux/arm64`.
- Tags: push→main = `:main` + `:main-<short-sha>`; release `vX.Y.Z` = `:X.Y.Z` + `:latest` (latest tracks releases, not main).
- Seed version **0.1.0** (pre-1.0).
- Conventional Commits; no attribution trailers.
- Publish only off non-PR events (PRs build but do not push).

## Task 4: CI workflow `build.yml`

**Files:**
- Create: `.github/workflows/build.yml`

**Interfaces:**
- Produces: on PR → `test` job only; on push to `main` → `test` then `image` (push `:main` + `:main-<sha>`); on release → `test` then `image` (push `:X.Y.Z` + `:latest`).

- [ ] **Step 1: Write the workflow**

Create `.github/workflows/build.yml`:

```yaml
name: Build and Test

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]
  release:
    types: [created]

permissions:
  contents: read
  packages: write

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: actions/setup-dotnet@v5
        with:
          dotnet-version: '10.0.x'
      - name: Check formatting
        run: dotnet format Inkshelf.sln --verify-no-changes
      - name: Run unit tests
        run: dotnet test tests/Inkshelf.Tests/Inkshelf.Tests.csproj

  image:
    needs: test
    if: github.event_name != 'pull_request'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: docker/setup-qemu-action@v3
      - uses: docker/setup-buildx-action@v3
      - name: Log in to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      - name: Compute tags
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/thomaslazar/inkshelf
          tags: |
            type=raw,value=main,enable={{is_default_branch}}
            type=sha,prefix=main-,enable={{is_default_branch}}
            type=semver,pattern={{version}}
            type=raw,value=latest,enable=${{ github.event_name == 'release' }}
      - name: Build and push
        uses: docker/build-push-action@v6
        with:
          context: .
          platforms: linux/amd64,linux/arm64
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

Note: `image` runs only when `github.event_name != 'pull_request'`, so on PRs there is no build/push at all (the metadata/semver tags require a default branch or a release tag, so building on PRs would produce no tags). The `test` job is the PR gate.

- [ ] **Step 2: Validate YAML locally**

Run: `python3 -c "import yaml,sys; yaml.safe_load(open('.github/workflows/build.yml')); print('yaml ok')"`
Expected: `yaml ok`.


## Task 5: Release skill

**Files:**
- Create: `.claude/skills/release/SKILL.md`

**Interfaces:**
- Produces: a human-gated `/release` skill adapted from abs-cli, publishing a container image instead of binaries.

- [ ] **Step 1: Write the skill**

Create `.claude/skills/release/SKILL.md` with frontmatter and the gated steps below. Base it on abs-cli's structure; swap the artifact.

Frontmatter:

```markdown
---
name: release
description: Cut a new Inkshelf release with human review gates — bumps version, writes changelog, opens a PR for CI, then tags and publishes the container image.
disable-model-invocation: true
allowed-tools:
  - Bash
  - Read
  - Write
  - Glob
  - Grep
  - Edit
  - AskUserQuestion
---
```

Body — the steps (each gate is mandatory; never skip):

1. **Preflight:** assert on `main`, clean tree, `git pull`; run
   `dotnet format Inkshelf.sln --verify-no-changes` and
   `dotnet test tests/Inkshelf.Tests/Inkshelf.Tests.csproj`. Determine version
   from conventional commits since last tag (`git describe --tags --abbrev=0`;
   any `feat:` → bump minor, else patch). **GATE: confirm version with the human.**
   (Note: no live smoke test yet — a seeded-ABS smoke gate is a future addition.)
2. **Branch + bump:** `git checkout -b release/vX.Y.Z`; `Edit` `<Version>` in
   `src/Inkshelf/Inkshelf.csproj` to `X.Y.Z`; `grep` to confirm; `dotnet build`.
   Commit `chore: bump version to X.Y.Z`.
3. **Changelog:** generate `release-notes.md` (Highlights + grouped conventional
   commits via `git log --oneline <lasttag>..HEAD`). **GATE: human reviews
   release-notes.md.** Prepend to `CHANGELOG.md` under the top header; commit
   `docs: add vX.Y.Z changelog entry`.
4. **PR → CI:** push branch, `gh pr create --base main`; poll for the run
   (`gh run list --branch release/vX.Y.Z`), `gh run watch <id> --exit-status`,
   report job results via `gh run view --json`. **GATE: human merges the PR.**
5. **Tag + Release:** `git checkout main && git pull`;
   `gh release create vX.Y.Z --title vX.Y.Z --notes-file release-notes.md`;
   `rm release-notes.md`. **GATE: confirm release created (show URL).**
6. **Wait for release CI:** find the `release`-event run, `gh run watch
   --exit-status`; confirm the `image` job pushed `:X.Y.Z` + `:latest`.
7. **Verify:** `docker buildx imagetools inspect ghcr.io/thomaslazar/inkshelf:X.Y.Z`
   and confirm both `linux/amd64` and `linux/arm64` are present. **GATE: human
   checks the GitHub Release page + the GHCR package page.**

Rules section (copy intent from abs-cli): never skip a gate; never proceed past
a failed check; clean up `release-notes.md`; CHANGELOG is the source of truth;
this skill may commit as part of its defined steps (overrides CLAUDE.md's
ask-before-commit rule).

Run: `python3 -c "import yaml; d=open('.claude/skills/release/SKILL.md').read().split('---')[1]; yaml.safe_load(d); print('frontmatter ok')"`
Expected: `frontmatter ok`.

- [ ] **Step 3: Commit**

```bash
git add .claude/skills/release/SKILL.md
git commit -m "chore: add gated release skill for container releases"
```
